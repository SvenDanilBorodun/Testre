# 04 — Environment Variables (Consolidated)

> **What this file is:** every env var consumed across all systems (Railway, Modal, Docker, GUI, .env files). Read when changing any configuration.
> Authoritative source per layer: see the layer file for that subsystem.

---

## 1. Where env vars live

| Surface | Storage | Loaded by | Lifetime |
|---|---|---|---|
| Railway FastAPI | Railway dashboard variables | `python-dotenv` + `os.environ.get()` at process start | Process lifetime; redeploy to update |
| Modal worker | `modal secret create edubotics-training-secrets --from-dotenv` | Modal injects at container start | Container lifetime (entire 7h max) |
| Modal MCP | `modal secret create mcp-edubotics --from-dotenv` | Modal injects at container start | ASGI app lifetime |
| Docker compose | `--env-file %LOCALAPPDATA%\EduBotics\.env` | Generated by `gui/app/config_generator.py` per startup | Compose run |
| Student React build | `REACT_APP_*` build args, baked into bundle | `npm run build` | Image build (immutable in shipped image) |
| Web React build (Railway) | Railway service variables → `Dockerfile.web` ARGs | Railway redeploys when changed | Image build |
| Windows GUI | `%LOCALAPPDATA%\EduBotics\.env` (read-only, used as compose --env-file) + a few `EDUBOTICS_*` overrides | `gui/app/constants.py` reads on startup | GUI process |
| Local dev (cloud_training_api) | `cloud_training_api/.env` (gitignored) | `dotenv` | Process |
| Local dev (modal_training) | `modal_training/.env` (gitignored, used to seed Modal Secret) | One-time `modal secret create` | n/a |

## 5. Docker compose — generated `.env` at `%LOCALAPPDATA%\EduBotics\.env`

Generated by `gui/app/config_generator.py:generate_env_file()` per startup. All values **double-quoted** (handles spaces in paths like `C:\Users\Max Muster\...`).

### Full mode

```
FOLLOWER_PORT="/dev/serial/by-id/usb-ROBOTIS_OpenRB-150_..."
LEADER_PORT="/dev/serial/by-id/usb-ROBOTIS_OpenRB-150_..."
CAMERA_DEVICE_1="/dev/video0"
CAMERA_NAME_1="gripper"
CAMERA_DEVICE_2="/dev/video2"
CAMERA_NAME_2="scene"
ROS_DOMAIN_ID=42         # 0–232, derived from machine UUID hash, fallback 30
REGISTRY="nettername"
```

### Cloud-only mode

```
# Cloud-only mode — no robot hardware connected.
FOLLOWER_PORT=""
LEADER_PORT=""
CAMERA_DEVICE_1=""
CAMERA_NAME_1="gripper"
CAMERA_DEVICE_2=""
CAMERA_NAME_2="scene"
ROS_DOMAIN_ID=42
REGISTRY="nettername"
```

| Var | Required | Used in | Notes |
|---|---|---|---|
| `FOLLOWER_PORT` | ✅ (full mode) | `docker-compose.yml:21`, `entrypoint_omx.sh` | udev-stable serial path |
| `LEADER_PORT` | ✅ (full mode) | `docker-compose.yml:22` | |
| `CAMERA_DEVICE_1`, `CAMERA_DEVICE_2` | ✅ (full mode) | `docker-compose.yml:23,25` | `/dev/videoN` |
| `CAMERA_NAME_1`, `CAMERA_NAME_2` | ✅ | `docker-compose.yml:24,26` | Default `gripper` / `scene`. **Must match policy `config.json` input_features at inference** |
| `ROS_DOMAIN_ID` | ✅ | compose env, ROS2 nodes | Per-machine to avoid LAN cross-talk. Default 30 (legacy) |
| `REGISTRY` | ✅ | `docker-compose.yml` image refs | `nettername` for production |

## 11. Cross-cutting checks before changing any var

1. **Does the same var live in multiple surfaces?** Yes for: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `HF_TOKEN`, `ROS_DOMAIN_ID`, `REGISTRY`. Bumping one without the others = drift. See [`20-operations.md`](20-operations.md) §1 for rotation procedures.
2. **Is it baked at build time?** All `REACT_APP_*` vars are baked into Docker images. Changing them = rebuild + push images.
3. **Does it require a redeploy?** Yes for: Railway (`railway up --detach`), Modal (`modal deploy modal_app.py`), Modal Secret (`modal secret create ... --from-dotenv`).
4. **Does the GUI write it?** Anything in the generated `.env` (full mode + cloud-only mode) is regenerated on every startup; manually editing `%LOCALAPPDATA%\EduBotics\.env` is wasted unless you also change the generator.
5. **Is it in `.gitignore`?** All `.env` files are gitignored. Don't commit secrets.

## 12. Local development setup

For working on `cloud_training_api/` locally:

```bash
cd robotis_ai_setup/cloud_training_api
cp .env.example .env  # if it exists
# Fill in: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, MODAL_TOKEN_*, HF_TOKEN, ALLOWED_ORIGINS=http://localhost:3000
pip install -r requirements.txt
uvicorn app.main:app --reload
```

For working on `modal_training/`:

```bash
cd robotis_ai_setup/modal_training
modal token new          # if not authenticated
modal serve modal_app.py # dev mode
# Or to update the Modal Secret:
# modal secret create edubotics-training-secrets --from-dotenv .env
```

For working on `physical_ai_manager/`:

```bash
cd physical_ai_tools/physical_ai_manager
cp .env.example .env  # if it exists
# Fill in: REACT_APP_SUPABASE_URL, REACT_APP_SUPABASE_ANON_KEY, REACT_APP_CLOUD_API_URL, REACT_APP_MODE=student
npm install
npm start
```

For working on `modal_mcp/`:

```bash
cd modal_mcp
# Update Modal Secret:
modal secret create mcp-edubotics --from-dotenv .env
modal serve mcp_server_stateless.py    # dev
# Test:
modal run -m mcp_server_stateless::test_tool --tool-name hf_model_info --tool-args-json '{"repo_id":"lerobot/act"}'
```
